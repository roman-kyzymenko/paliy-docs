# Політика конфіденційності — застосунок Віджет Палій

[English version](paliy-docs/privacy-policy/en)
[Українська версія]

_Останнє оновлення: 2025-11-19_

Віджет Палій — це приватний, неурядовий проєкт. Він не пов’язаний з жодним державним органом, постачальником послуг або муніципальною організацією в Україні.  
Застосунок надає зручний спосіб переглядати загальнодоступні графіки відключень та пов’язані дані.

Ця Політика конфіденційності описує, як застосунок обробляє дані на iOS та Android.

---

## 1. Загальний огляд

Віджет Палій:

- **не** збирає персональні дані
- **не** створює облікові записи
- **не** збирає дані про місцезнаходження
- **не** використовує аналітику або трекінг
- **не** показує рекламу
- **не** обробляє платежі
- використовує лише загальнодоступні дані, які надаються бекендом розробника
- спілкується лише з власними API-серверами

Ваші дані ніколи не продаються, не передаються третім сторонам і не використовуються для відстеження.

## 2. Дані, які ми збираємо

### Ми НЕ збираємо:

- Персональну інформацію (ім’я, email, номер телефону)
- Ідентифікатори пристрою (Device ID, Advertising ID)
- Дані про місцезнаходження
- Контакти, фотографії, медіа чи файли
- Аналітику використання
- Будь-які дані, що можуть ідентифікувати вас

### Ми ОБРОБЛЯЄМО:

#### 2.1. Технічні дані запитів (стандартні HTTP-логи)

Коли застосунок або віджет надсилає запит до бекенду, сервер отримує технічні метадані, які ваш пристрій передає автоматично:

- IP-адреса
- User agent (модель пристрою, версія ОС)
- Час запиту

Ця інформація використовується лише для безпеки сервера, забезпечення доступності та лімітів запитів.  
Вона **не** пов’язується з жодним профілем користувача і **не** використовується для відстеження.

#### 2.2. Загальнодоступні дані про відключення

Застосунок показує:

- графіки відключень електроенергії
- назви вулиць для ручного пошуку

Ці набори даних є публічними і не містять персональної інформації.

## 3. Як працює застосунок

### 3.1. Взаємодія з бекендом

Застосунок використовує власний бекенд розробника для отримання:

- графіків відключень
- списків вулиць для пошуку
- даних для оновлення віджета

Приватні дані користувачів не передаються.

### 3.2. Поведінка віджета (iOS та Android)

Віджет може:

- періодично оновлюватися відповідно до правил ОС

Усі запити віджета містять лише публічні дані.  
Віджети **не збирають** і **не зберігають** персональних даних.

### 3.3. Зовнішні посилання

Застосунок може містити посилання на зовнішні сервіси, такі як Telegram-бот розробника.  
Взаємодія з такими сервісами регулюється їхніми власними політиками конфіденційності.

## 4. Push-повідомлення / Тихі фонові повідомлення

Застосунок **не** надсилає користувачам видимі сповіщення.  
Однак, версії для iOS та Android використовують **фонові повідомлення** для оновлення даних віджета або ініціювання його оновлення.

Такі повідомлення:

- **не** містять персональної інформації
- **не** прив’язані до окремих пристроїв (використовують топіки)
- **не** показують банери, значки чи звуки
- використовуються лише для актуалізації даних віджета

### 4.1. iOS (фонові повідомлення APNs)

Версія для iOS використовує Apple Push Notification Service (APNs) **лише** для:

- отримання фонових повідомлень “content-available”
- оновлення даних віджета у фоновому режимі

### 4.2. Android (фонові повідомлення FCM)

Версія для Android використовує Firebase Cloud Messaging (FCM) **лише** для:

- отримання фонових повідомлень за топіками
- оновлення контенту віджета у фоновому режимі

На жодній платформі не відображаються видимі сповіщення.

## 5. Платформно-специфічна інформація

### 5.1. iOS

Версія для iOS може використовувати такі системні можливості:

- **WidgetKit** — для відображення графіків у віджеті
- **Фонове оновлення застосунку** — для періодичного оновлення
- **Тихі push-повідомлення (APNs)** — для тригерів оновлення
- **App Groups** — для зберігання налаштувань віджета (без персональних даних)

Жодні персональні дані не зберігаються в App Groups.

### 5.2. Android

Версія для Android може використовувати:

- **App Widgets Framework** — для відображення графіків
- **Фонові оновлення** — що контролюються ОС Android
- **Firebase Cloud Messaging (FCM)** — для тихих фонових оновлень
- **SharedPreferences** — для зберігання налаштувань віджета (без персональних даних)

Жодні персональні або унікальні ідентифікуючі дані не зберігаються.
