# Політика конфіденційності — застосунок Віджет Палій

[English version](en)
[Українська версія]

_Останнє оновлення: 2025-11-19_

Віджет Палій — це приватний, неурядовий проєкт. Він не пов’язаний з жодним державним органом, постачальником послуг або муніципальною організацією в Україні.  
Застосунок надає зручний спосіб переглядати загальнодоступні графіки відключень та пов’язані з ними дані.

Ця Політика конфіденційності описує, як застосунок обробляє дані на iOS та Android.

---

## 1. Загальний огляд

Віджет Палій:

- **не** збирає персональні дані
- **не** створює облікові записи
- **не** збирає дані про місцезнаходження
- **не** використовує аналітику чи трекінг
- **не** показує рекламу
- **не** обробляє платежі
- використовує лише загальнодоступні дані, що надаються бекендом розробника
- взаємодіє лише з власними API-серверами

Ваші дані ніколи не продаються, не передаються третім сторонам і не використовуються для відстеження.

---

## 2. Дані, які ми збираємо

### Ми НЕ збираємо:

- Персональну інформацію (ім’я, email, номер телефону)
- Ідентифікатори пристрою (Device ID, Advertising ID)
- Дані про місцезнаходження
- Контакти, фотографії, медіа чи файли
- Аналітику використання
- Будь-які дані, що можуть ідентифікувати користувача

### Ми ОБРОБЛЯЄМО:

#### 2.1. Технічні дані запитів (стандартні HTTP-логи)

Коли застосунок або віджет надсилає запит до бекенду, сервер отримує технічні метадані, які ваш пристрій передає автоматично:

- IP-адресу
- User agent (модель пристрою, версія ОС)
- Час запиту

Ця інформація використовується лише для безпеки сервера, забезпечення доступності та обмеження навантаження.  
Вона **не** пов’язується з жодним профілем користувача і **не** використовується для відстеження.

#### 2.2. Загальнодоступні дані про відключення

Застосунок відображає:

- графіки відключень електроенергії
- списки вулиць для ручного пошуку

Ці набори даних є публічними та не містять персональної інформації.

## 3. Як працює застосунок

### 3.1. Взаємодія з бекендом

Застосунок використовує власний бекенд розробника для отримання:

- графіків відключень
- списків вулиць для пошуку
- даних для оновлення віджета

Приватні дані користувачів не передаються.

### 3.2. Поведінка віджета (iOS та Android)

Віджет може:

- оновлюватися періодично відповідно до правил ОС
- виконувати фонові вибірки даних
- запитувати оновлену інформацію з бекенду
- відображати актуальні дані про відключення

Усі запити віджета містять лише загальнодоступні дані.  
Віджет **не зберігає** і **не збирає** персональних даних.

### 3.3. Зовнішні посилання

Застосунок може містити посилання на зовнішні сервіси, такі як Telegram-бот розробника.  
Взаємодія з такими сервісами регулюється їхніми власними політиками конфіденційності.

## 4. Push-повідомлення / Тихі фонові повідомлення

Застосунок **не** надсилає користувачам видимих сповіщень.  
Однак версії для iOS та Android використовують **тихі фонові повідомлення** для оновлення даних віджета або запуску процесу оновлення.

Щоб доставити такі повідомлення, операційна система або сервіс Firebase може згенерувати **непрозорий токен пристрою** (Apple) або **реєстраційний токен** (Google):

- Ці токени необхідні для маршрутизації фонових повідомлень.
- Вони **не** ідентифікують користувача персонально.
- Вони **не** використовуються, **не** зберігаються та **не** обробляються розробником.
- Вони **не** передаються на бекенд розробника.
- Вони можуть бути передані Apple або Google виключно для доставки повідомлення.

Тихі фонові повідомлення:

- **не** містять персональної інформації
- **не** є орієнтованими на конкретний пристрій (використовуються топіки)
- **не** показують банери, значки чи звуки
- використовуються лише для оптимізації часу оновлення віджета

---

## 4.1. iOS (тихі повідомлення APNs)

Версія для iOS використовує Apple Push Notification Service (APNs) **лише** для:

- отримання тихих повідомлень “content-available”
- оновлення даних віджета у фоновому режимі

iOS може згенерувати токен пристрою для маршрутизації цих повідомлень.  
Цей токен повністю обробляється інфраструктурою Apple.  
Розробник **не** отримує доступу до цього токена, не використовує його та не зберігає.

---

## 4.2. Android (тихі повідомлення FCM)

Версія для Android використовує Firebase Cloud Messaging (FCM) **лише** для:

- отримання тихих повідомлень за топіками
- автоматичного оновлення контенту віджета

Firebase може згенерувати реєстраційний токен як частину механізму FCM.  
Цей токен повністю управляється Google і не використовується розробником.

Жодних видимих сповіщень на жодній платформі не відображається.

---

## 5. Платформно-специфічна інформація

### 5.1. iOS

Версія для iOS може використовувати такі функції системи:

- **WidgetKit** — відображає дані у віджеті
- **Фонове оновлення застосунку** — для періодичних оновлень
- **Тихі push-повідомлення (APNs)** — для тригерів оновлення
- **App Groups** — для зберігання налаштувань віджета (без персональних даних)

Жодні персональні дані в App Groups не зберігаються.

### 5.2. Android

Версія для Android може використовувати:

- **App Widgets Framework** — для відображення графіків
- **Фонові оновлення**, що контролюються ОС Android
- **Firebase Cloud Messaging (FCM)** — для тихих фонових оновлень
- **SharedPreferences** — для зберігання налаштувань віджета (без персональних даних)

Жодні персональні або унікальні ідентифікувальні дані не зберігаються.
